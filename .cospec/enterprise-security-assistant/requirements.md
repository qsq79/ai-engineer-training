# 企业级安全智能助手 - 需求文档

**版本**: v1.0
**创建日期**: 2026-03-08

---

## 1. 项目概述

### 1.1 项目背景
面向企业级SaaS安全平台，提供多Agent协同的智能安全分析助手，通过专业化分工和协作，为安全运营团队提供全方位的智能支持。

### 1.2 核心价值
- 提升安全运营效率，降低误报率
- 加速问题诊断，提供操作指引
- 数据可视化，决策支持
- 自动化合规检查，快速生成报告

---

## 2. 核心功能需求

### 2.1 意图识别Agent（P0）

**功能描述**：理解用户自然语言查询，识别意图类型，提取关键参数，路由到合适的业务Agent。

**用户故事**：作为一个安全分析师，我可以通过自然语言描述我的问题，系统能自动理解我的意图并调用正确的分析工具，无需手动选择复杂的菜单。

**功能子需求**：
- [R-1.1] 自然语言理解：支持文本清洗、分词、停用词过滤
- [R-1.2] 意图分类：支持5种意图类型（query_diff、scoring_explanation、threat_analysis、compliance_check、knowledge_search）
- [R-1.3] 参数提取：支持提取report_id、metric_name、score、time_range、threat_type等参数
- [R-1.4] 上下文管理：支持会话上下文补充和历史记录
- [R-1.5] 置信度计算：计算意图分类的置信度，支持阈值配置
- [R-1.6] Agent路由：根据意图类型和置信度路由到目标Agent
- [R-1.7] 响应时间：意图识别响应时间 ≤ 500ms
- [R-1.8] 准确率：意图识别准确率 ≥ 90%

**验收标准**：
- 能正确识别5种意图类型
- 能正确提取常用参数
- 意图识别准确率 ≥ 90%
- 响应时间 ≤ 500ms

---

### 2.2 工作流协调Agent（P0）

**功能描述**：协调多个Agent协作，管理任务流程，处理任务依赖、状态管理和结果聚合。

**用户故事**：作为一个安全工程师，我需要执行复杂的攻击溯源分析，系统可以自动协调多个Agent并行工作，汇总分析结果并生成综合报告。

**功能子需求**：
- [R-2.1] 工作流解析：解析工作流配置，创建任务，建立依赖关系
- [R-2.2] 任务调度：支持拓扑排序，按依赖顺序调度任务
- [R-2.3] 并行执行：支持无依赖任务的并行执行
- [R-2.4] 依赖管理：管理任务之间的依赖关系，确保依赖满足后才执行
- [R-2.5] 状态管理：跟踪任务状态（PENDING、RUNNING、COMPLETED、FAILED、TIMEOUT）
- [R-2.6] 超时处理：支持任务超时检测和处理
- [R-2.7] 错误重试：支持失败任务的自动重试，可配置重试次数和策略
- [R-2.8] 结果聚合：收集所有任务结果，生成汇总摘要
- [R-2.9] 工作流模式：支持串行、并行、迭代三种协作模式
- [R-2.10] 性能要求：任务调度延迟 ≤ 100ms

**验收标准**：
- 能正确执行串行工作流
- 能正确执行并行工作流
- 能正确处理任务依赖
- 能正确处理超时和错误

### 2.3 日志查询Agent（P0）

**功能描述**：解释日志数据差异，生成查询SQL，提供数据对账支持。

**用户故事**：作为一个安全分析师，当我发现月报数据与我的查询结果不一致时，系统可以自动分析差异原因，生成正确的SQL语句，并解释数据差异的原因。

**功能子需求**：
- [R-3.1] 条件对比：对比月报查询条件和用户查询条件，识别差异
- [R-3.2] SQL生成：基于差异分析生成正确的SQL查询语句
- [R-3.3] 差异解释：用自然语言解释数据差异的原因
- [R-3.4] 月报数据获取：根据月报ID获取月报配置和查询条件
- [R-3.5] 用户查询条件获取：获取用户历史查询条件
- [R-3.6] 响应时间：响应时间 ≤ 5s

**验收标准**：
- 能正确识别查询条件差异
- 能生成正确的SQL语句
- 能解释数据差异的原因
- 响应时间 ≤ 5s

### 2.4 评分解读Agent（P0）

**功能描述**：解读安全评分，解释评分逻辑，提供评分明细和改进建议。

**用户故事**：作为一个安全经理，我看到系统给出的安全评分是75分，我需要了解这个分数是如何计算的，哪些方面得分低，如何改进。

**功能子需求**：
- [R-4.1] 文档检索：基于向量数据库检索评分相关文档和知识
- [R-4.2] 评分解释：用自然语言解释评分逻辑和计算方法
- [R-4.3] 明细展示：展示评分明细，包括各维度的得分和权重
- [R-4.4] 改进建议：基于低分项提供改进建议
- [R-4.5] 响应时间：响应时间 ≤ 3s

**验收标准**：
- 能正确解释评分逻辑
- 能展示评分明细
- 能提供改进建议
- 响应时间 ≤ 3s

### 2.5 威胁分析Agent（P0）

**功能描述**：分析威胁情报，识别潜在威胁，构建攻击图，生成威胁分析报告。

**用户故事**：作为一个安全运维工程师，我需要分析一个攻击事件，系统可以查询威胁情报库，分析攻击模式，构建攻击图，识别威胁链路，生成威胁分析报告。

**功能子需求**：
- [R-5.1] 威胁情报查询：支持IP、域名、Hash、IOC等多维度威胁情报查询
- [R-5.2] 攻击模式分析：识别攻击阶段，分析TTP（战术、技术、程序），映射CVE漏洞
- [R-5.3] 攻击图构建：构建攻击图，创建节点和边，计算权重
- [R-5.4] 威胁链路识别：追踪威胁链路，识别因果关系，分析时间序列
- [R-5.5] 威胁评估：评估威胁等级，分析影响范围，提供处置建议
- [R-5.6] MITRE ATT&CK映射：将攻击行为映射到MITRE ATT&CK框架
- [R-5.7] 实体提取：从日志中提取IP、域名、用户、进程等实体
- [R-5.8] 关系提取：提取网络连接、进程启动、文件访问等关系
- [R-5.9] 响应时间：响应时间 ≤ 10s

**验收标准**：
- 能正确查询威胁情报
- 能正确分析攻击模式
- 能正确构建攻击图
- 响应时间 ≤ 10s

**功能子需求**：

- [R-6.2] 等保2.0规则：实现等保2.0二级规则（访问控制、审计日志、数据保护等）

- [R-6.4] 配置检查：检查系统配置、安全策略、权限配置
- [R-6.5] 日志审计：收集日志，分析日志，检测异常
- [R-6.6] 规则验证：执行规则匹配，判断状态，评级严重性
- [R-6.7] 结果汇总：计算评分，汇总问题，计算合规率
- [R-6.8] 报告生成：生成合规报告，包括图表和格式化输出
- [R-6.9] 响应时间：响应时间 ≤ 30s

**验收标准**：
- 能正确执行合规检查
- 能正确生成合规报告
- 响应时间 ≤ 30s

## 8. 附录

### 8.1 术语表
- **Agent**：智能代理，负责执行特定任务的AI组件
- **工作流**：由多个Agent协作完成的任务流程
- **多租户**：一个系统实例服务多个客户，每个客户的数据完全隔离
- **RBAC**：基于角色的访问控制（Role-Based Access Control）
- **LLM**：大语言模型（Large Language Model）
- **RAG**：检索增强生成（Retrieval-Augmented Generation）
- **MITRE ATT&CK**：对抗战术和技术知识库
