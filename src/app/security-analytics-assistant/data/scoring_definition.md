# 安全评分计算方法（v2.1）

## 评分维度及权重

- **攻击次数（30%）**：统计周期内检测到的总攻击次数
  - 0-1000次：满分30
  - 1000-5000次：20分
  - 5000次以上：10分

- **攻击成功次数（30%）**：实际成功的攻击次数
  - 0次：30分
  - 1-5次：20分
  - 5次以上：10分

- **告警数量（20%）**：生成的告警总数
  - 0-50次：20分
  - 51-100次：15分
  - 100次以上：5分

- **漏洞风险（10%）**：根据高危漏洞数量计算
  - 0个高危漏洞：10分
  - 1-2个高危漏洞：5分
  - 3个以上高危漏洞：0分

- **弱密码（5%）**：存在弱密码的资产比例
  - 0%资产：5分
  - 1-10%资产：3分
  - 10%以上资产：0分

- **暴露面风险（5%）**：开放的高危端口数量
  - 0个高危端口：5分
  - 1-2个高危端口：3分
  - 3个以上高危端口：0分

## 计算公式

总分 = 攻击次数得分 + 攻击成功得分 + 告警得分 + 漏洞风险得分 + 弱密码得分 + 暴露面得分

### 示例计算

**示例 1：安全评分 78 分**

- 攻击次数（满分30）：攻击次数 2500 次 → 得分 20 分
- 攻击成功（满分30）：攻击成功 3 次 → 得分 20 分
- 告警数量（满分20）：告警 85 次 → 得分 15 分
- 漏洞风险（满分10）：高危漏洞 2 个 → 得分 5 分
- 弱密码（满分5）：弱密码比例 2% → 得分 3 分
- 暴露面（满分5）：高危端口 0 个 → 得分 5 分

总分：20 + 20 + 15 + 5 + 3 + 5 = 68 分

**示例 2：安全评分 92 分**

- 攻击次数（满分30）：攻击次数 500 次 → 得分 30 分
- 攻击成功（满分30）：攻击成功 0 次 → 得分 30 分
- 告警数量（满分20）：告警 30 次 → 得分 20 分
- 漏洞风险（满分10）：高危漏洞 0 个 → 得分 10 分
- 弱密码（满分5）：弱密码比例 0% → 得分 5 分
- 暴露面（满分5）：高危端口 0 个 → 得分 5 分

总分：30 + 30 + 20 + 10 + 5 + 5 = 100 分

## 常见问题

### 为什么分数低？

评分低通常有以下原因：
1. 攻击次数较多（超过5000次，得分仅10分）
2. 攻击成功次数较多（超过5次，得分仅10分）
3. 告警数量过多（超过100次，得分仅5分）
4. 存在高危漏洞（超过2个，得分为0）
5. 弱密码资产比例高（超过10%，得分为0）
6. 开放高危端口较多（超过3个，得分为0）

### 如何提高评分？

提高安全评分的建议：
1. **减少攻击次数**：加强入侵防护系统，减少被攻击次数
2. **阻止攻击成功**：提升防火墙规则，阻止攻击成功
3. **减少告警**：优化告警阈值，减少无效告警
4. **修复漏洞**：及时修复高危漏洞，减少安全风险
5. **强化密码策略**：强制使用强密码，减少弱密码资产
6. **关闭高危端口**：关闭不必要的高危端口，减少暴露面
