# 智能安全数据分析助手

基于 LangChain 构建的智能安全数据分析助手，支持复杂日志查询解释和安全评分解读两大核心场景。

## 项目背景

### 业务场景

本项目基于真实的 SaaS 安全平台业务场景，解决两个高频客户痛点：

1. **日志数据对账问题**：客户在月报中看到"攻击日志top10"，但在日志检索中心自己查询时，数量对不上。因为后台查询时过滤了资产组IP、资源组IP等复杂条件，客户不知道这些隐藏的过滤逻辑。

2. **安全评分解读问题**：月报给客户打了一个安全评分（比如78分），但没有说明这个分数是如何计算的。客户问"为什么我只有78分"，客服需要翻阅大量文档才能回答。

### 技术方案

本项目整合了 LLM 应用的两种典型模式：

1. **Agent 模式（场景一）**：通过多工具协同完成复杂查询条件对比和 SQL 生成
2. **RAG 模式（场景二）**：通过文档检索增强，结合具体分数给出个性化解释

## 功能特性

### 场景一：复杂日志查询解释

- 🔍 **条件对比**：自动对比月报查询条件和用户查询条件
- 📝 **SQL 生成**：生成可在日志中心直接执行的 SQL
- 💡 **智能解释**：解释数量差异的具体原因（资产组过滤、IP 排除等）

### 场景二：安全评分解读

- 📚 **文档检索**：实时检索评分定义文档
- 🎯 **精准解释**：结合用户的具体分数给出个性化分析
- 📊 **明细展示**：展示各维度得分和扣分原因

## 技术栈

| 技术组件 | 说明 |
|---------|------|
| LangChain | Agent 和 RAG 框架 |
| LangGraph | Agent 运行时 |
| OpenAI | LLM 模型 |
| Chroma/FAISS | 向量数据库（RAG） |
| ClickHouse | 实际日志数据源（可选） |

## 快速开始

### 1. 安装依赖

```bash
cd src/app/security-analytics-assistant
pip install -r requirements.txt
```

### 2. 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env` 文件，填入你的 API Key。

### 3. 运行程序

#### 交互式模式（推荐）

```bash
python main.py
```

#### 场景一：日志查询解释

```bash
python main.py --mode query
```

#### 场景二：评分解读

```bash
python main.py --mode scoring
```

## 使用示例

### 场景一示例

**用户问题**：为什么我查的SQL注入攻击数量比月报少？

**Agent 回答**：
```
我查了一下，月报里的SQL注入攻击统计有两个过滤条件：
1. 只统计了资产组A和B的资产（您的查询可能统计了所有资产）
2. 排除了内网IP段192.168.1.0/24的告警

这是月报使用的完整查询条件，您可以复制到日志中心重新查询：
SELECT COUNT(*) FROM attack_logs 
WHERE attack_type='sql_injection' 
AND asset_group IN ('group_A', 'group_B')
AND src_ip NOT LIKE '192.168.1.%'
AND timestamp BETWEEN '2026-02-01' AND '2026-02-28'
```

### 场景二示例

**用户问题**：这个月的安全评分78分，是怎么算的？

**Agent 回答**：
```
根据我们的评分文档（v2.1版），安全评分由6个维度加权计算：

您这个月78分，主要扣分项是：
1. 攻击成功次数：有3次成功攻击（扣了10分）
2. 高危漏洞：存在2个高危漏洞（扣了5分）

具体明细：
- 攻击次数（满分30）：您得了28分（攻击次数在1000-5000区间）
- 攻击成功（满分30）：您得了20分（3次成功攻击）
- 告警数量（满分20）：您得了18分
- 漏洞风险（满分10）：您得了5分（2个高危漏洞）
- 弱密码（满分5）：满分5分
- 暴露面（满分5）：满分5分
```
